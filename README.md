##　このプロジェクトについて
c++を用いて初めてゲーム制作を行いました。enum class を用いた状態管理や UI 配置などにこだわりました。個人開発ながら保守性や可読性を意識して実装しました。

## ■ 1. ゲーム概要

### タイトル

MyMineSweeper

### コンセプト

- ### 詰まない運要素を排除したマインスイーパー
  ソルバーを導入することで詰むことのない盤面のみを生成し、運要素を排除したマインスイーパーを作成しました。この機能を応用し、あまり得意でない方向けのヒント機能も実装しました。

### アピールポイント

C++と SFML ライブラリを用いて、ゲームのコアロジックから UI 描画までをフルスクラッチで実装。特に、盤面の解の存在を保証するソルバーアルゴリズムの設計に注力しました。

## ■ 2. システム設計と実装の工夫

### GLSL シェーダの活用

モダンな UI デザインを実現するために、SFML の標準機能に加え GLSL フラグメントシェーダを導入。
　１．メインメニューのグレーの斜線の直線の方程式を求める。
　２．シェーダー側でピクセル座標と直線を比較し、斜線の片側のピクセルのみを描画することで斜線部で色の変更を実装。

### 詰まない盤面生成

プレイヤーが初回クリック時に
　１．生成：初回クリック地点の周囲の地雷数が０であることを保証した盤面を生成。
　２．テスト：盤面をコピーし、Solver クラスに渡し論理的に最後まで解けるかを検証。
　３．1,2 の条件を満たした盤面のみ使用することで初手のマスが連鎖的に解放される爽快感と途中で詰んでしまうという理不尽さの解消を両立させた。

## ■ 3. 使用技術

- **使用言語**：C++
- **開発環境**：Visual Studio Code（MSYS2 + UCRT64 環境）
- **ライブラリ**：SFML（Simple and Fast Multimedia Library）

## ■ 4. 操作方法・ビルド方法（MSYS2 UCRT64 環境で実行）

```bash
g++ src/main.cpp src/Game.cpp src/Field.cpp src/GameUI.cpp src/GameRenderer.cpp src/SoundManager.cpp src/Solver.cpp -o MyMinesweeper -lsfml-graphics -lsfml-window -lsfml-system -lsfml-audio
実行方法
bash
./MyMinesweeper.exe
※ .exe をダブルクリックしても動作しない場合があります。MSYS2のターミナルから実行してください。

操作方法
左クリック：マスを開く
右クリック：旗を立てる

ボタン説明
Easy,Normal,Hard：メインメニュー画面でそれぞれのボタンに応じた難易度のボタンを押すことでゲーム開始

Menu：プレイ時にウィンドウ右上のメニューアイコンを押すことでメニュー画面に移動

Hint：メニューアイコン左の電球のアイコン。現在の盤面から論理的に開けることが確定している安全なマスを1つだけハイライトします。（※ヒントは、プレイヤーが旗を立てて状況を更新した後など、論理的に解ける手が存在する場合にのみ表示されます）

Continue、Finish：メニュー画面時にContinueボタンで元の状態に戻り、Finishボタンでメインメニュー画面に移動

GoTitle：ゲームオーバー時やクリア時にウィンドウ右上の「GoTitle」を押すことでタイトル画面に移動

```
